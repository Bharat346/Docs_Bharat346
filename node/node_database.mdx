
## Introduction

Node.js is widely used for backend development, and databases play a crucial role in building scalable applications. This guide explores various database options and how to integrate them with Node.js.

## Types of Databases

### 1. Relational Databases (SQL)
- **Examples:** MySQL, PostgreSQL, SQLite
- **Features:** Structured data, ACID compliance, strong relationships.
- **Usage:** Suitable for applications requiring complex queries and transactions.

### 2. NoSQL Databases
- **Examples:** MongoDB, Firebase, CouchDB
- **Features:** Flexible schema, high scalability, document-based storage.
- **Usage:** Ideal for applications handling unstructured or semi-structured data.

### 3. In-Memory Databases
- **Examples:** Redis, Memcached
- **Features:** High-speed performance, key-value storage.
- **Usage:** Caching, real-time data storage.

## Setting Up a Database in Node.js

### 1. Installing Dependencies
Depending on the database, install the required package:

#### MySQL
```sh
npm install mysql2
```

#### MongoDB
```sh
npm install mongoose
```

#### PostgreSQL
```sh
npm install pg
```

### 2. Connecting to a Database

#### Example: Connecting to MongoDB using Mongoose
```js
import mongoose from "mongoose";

const connectDB = async () => {
  try {
    await mongoose.connect("mongodb://localhost:27017/mydatabase", {
      useNewUrlParser: true,
      useUnifiedTopology: true,
    });
    console.log("✅ MongoDB Connected");
  } catch (error) {
    console.error("Error connecting to MongoDB:", error);
  }
};

connectDB();
```

#### Example: Connecting to MySQL
```js
import mysql from "mysql2";

const connection = mysql.createConnection({
  host: "localhost",
  user: "root",
  password: "password",
  database: "mydatabase",
});

connection.connect((err) => {
  if (err) {
    console.error("Error connecting to MySQL:", err);
    return;
  }
  console.log("✅ MySQL Connected");
});
```

## Performing CRUD Operations

### MongoDB CRUD Example
#### Create (Insert)
```js
const Blog = mongoose.model("Blog", new mongoose.Schema({ title: String, content: String }));
await Blog.create({ title: "First Blog", content: "This is my first blog post." });
```

#### Read (Find)
```js
const blogs = await Blog.find();
console.log(blogs);
```

#### Update
```js
await Blog.updateOne({ title: "First Blog" }, { content: "Updated content." });
```

#### Delete
```js
await Blog.deleteOne({ title: "First Blog" });
```

## Conclusion
Integrating a database with Node.js depends on the application's needs. SQL databases are best for structured data, NoSQL databases for flexible data models, and in-memory databases for caching. Choose the right database and ensure proper connections and operations for efficient backend development.
